# 技能系统 — 代理自我进化

## 1. 核心思想

技能（Skills）是 Hermes Agent **自我进化**的关键机制。它们是**可复用的过程性知识**——不是"用户喜欢什么颜色"这样的声明性记忆，而是"如何用 Playwright 抓取动态网页"这样的操作性知识。

与记忆的区别：

| 维度 | 记忆 (Memory) | 技能 (Skill) |
|---|---|---|
| **内容性质** | 声明性：谁、什么、何时 | 过程性：怎么做 |
| **组织结构** | 按类型分类（user/project/feedback） | 按领域分类（software-dev/creative/research） |
| **生命周期** | 随对话自然积累 | 由 Agent 主动创建、编辑、删除 |
| **可分享性** | 用户私有 | 可通过 Skills Hub 公开/分享 |
| **格式** | 自由文本（Markdown） | 结构化（SKILL.md + frontmatter + 附属文件） |

## 2. 技能存储结构

```
~/.hermes/skills/         # 用户私有技能（agent 可以创建/修改）
  ├── my-skill/
  │   ├── SKILL.md        # 核心文件：YAML frontmatter + Markdown 正文
  │   ├── references/     # 参考资料
  │   ├── templates/      # 代码模板
  │   ├── scripts/        # 可执行脚本
  │   └── assets/         # 静态资源
  └── another-skill/
      └── SKILL.md

skills/                    # 项目内置技能（只读，随代码发布）
  ├── software-development/
  │   ├── git-workflow/
  │   │   └── SKILL.md
  │   └── api-design/
  │       └── SKILL.md
  ├── creative/
  ├── research/
  └── ... (25 个分类)

~/.hermes/skills-hub/      # Skills Hub 安装的技能（只读，由 hub 管理）
```

## 3. SKILL.md 格式

### 3.1 Frontmatter

```yaml
---
name: git-workflow
description: Branch management, commit conventions, PR workflow
when_to_use: When working with git branches, creating commits, or opening PRs
platforms: [linux, macos, windows]    # 可选：平台限制
disabled: false                        # 可选：是否禁用
---
```

### 3.2 正文

技能正文是 LLM 在查阅技能时看到的完整内容——步骤、示例、注意事项：

```markdown
# Git Workflow

## Steps
1. Always create a feature branch from `main`
2. Use conventional commit messages
3. ...

## Examples
### Creating a feature branch
\`\`\`bash
git checkout -b feature/my-feature
\`\`\`

## Common Pitfalls
- Never force push to main
- ...
```

## 4. 技能工具链

### 4.1 工具一览

| 工具 | 文件 | 用途 |
|---|---|---|
| `skills_list` | `tools/skills_tool.py` (~54K) | 列出所有可用技能，支持搜索/过滤 |
| `skill_view` | `tools/skills_tool.py` | 查看技能的完整 SKILL.md 内容 |
| `skill_manage` | `tools/skill_manager_tool.py` (~30K) | 创建/编辑/删除/修补技能 |

### 4.2 skill_manage 操作

| 操作 | 说明 |
|---|---|
| **create** | 创建新技能：生成 SKILL.md + 目录结构 |
| **edit** | 完整替换 SKILL.md 内容 |
| **patch** | 目标性查找替换（修改某个章节） |
| **delete** | 删除用户技能（内置和 Hub 技能不可删除） |
| **write_file** | 添加附属文件（references/templates/scripts/assets） |
| **remove_file** | 删除附属文件 |

### 4.3 技能安全扫描

```python
# 从 Skills Hub 安装的技能总是经过安全扫描
# Agent 自己创建的技能只在 skills.guard_agent_created 开启时扫描
from tools.skills_guard import scan_skill, should_allow_install

# 扫描规则包括：
#   - 可疑的 shell 命令（curl | bash, eval, 等）
#   - 信息窃取模式
#   - 文件系统遍历
#   - 网络外连
```

## 5. 技能发现与加载流程

### 5.1 技能索引构建

```python
def build_skills_system_prompt(platform, disabled_skills):
    """构建注入系统提示词的技能索引"""
    skills = []
    for skill_dir in get_all_skills_dirs():  # 内置 + 用户 + Hub
        for skill_path in iter_skill_index_files(skill_dir):
            fm = parse_frontmatter(skill_path)
            if not skill_matches_platform(fm, platform):
                continue
            if fm['name'] in disabled_skills:
                continue
            skills.append({
                "name": fm['name'],
                "description": fm['description'],
                "when_to_use": fm.get('when_to_use', ''),
            })
    return format_skills_index(skills)
```

### 5.2 多来源优先级

当多个来源有同名技能时，优先级为：

1. **用户技能**（`~/.hermes/skills/`）— 最高优先级，可以覆盖内置/Hub
2. **Skills Hub 技能**（`~/.hermes/skills-hub/`）
3. **内置技能**（`skills/`）

## 6. Skills Hub — 技能市场

`tools/skills_hub.py`（约 119K）实现了完整技能市场：

### 6.1 功能

| 功能 | 说明 |
|---|---|
| **浏览** | 搜索 GitHub 上的公开技能仓库 |
| **安装** | 从 GitHub 克隆 + 安全扫描 + 安装到 `~/.hermes/skills-hub/` |
| **更新** | 检查已安装技能的更新（git pull） |
| **卸载** | 移除技能目录 |
| **发布** | Agent 可以将自己的技能推送到 Skills Hub |
| **索引缓存** | 本地缓存 Hub 索引以减少 GitHub API 调用 |

### 6.2 安装流程

```
1. 用户/Agent 请求安装 skill-name
2. 查询索引缓存 → 找到 GitHub 仓库 URL
3. git clone 到临时目录
4. scan_skill() 安全扫描
5. 检查依赖（需要的环境变量、工具等）
6. 复制到 ~/.hermes/skills-hub/skill-name/
7. 刷新技能索引缓存
```

## 7. 自我改进循环

这是 Hermes 最独特的特性——Agent 可以在使用中改进技能：

```
1. Agent 发现某项任务做得好 → 创建技能
2. 下次遇到类似任务 → 查阅并使用技能
3. 发现技能有不足 → 用 skill_manage patch 改进
4. 改进后的技能使下次执行更高效
5. 好的技能可以发布到 Skills Hub → 帮助其他用户
```
