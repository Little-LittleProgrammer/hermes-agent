# 插件系统与扩展机制

## 1. 概述

Hermes Agent 提供三层扩展机制，从简单到强大：

| 机制 | 复杂度 | 说明 |
|---|---|---|
| **插件** | 简单 | Python 包，提供 hook 函数 |
| **MCP 工具** | 中等 | 外部 MCP 服务器提供的工具 |
| **ACP 集成** | 复杂 | IDE/编辑器的深度集成协议 |

## 2. 插件系统

### 2.1 插件结构

```
~/.hermes/plugins/             # 用户插件目录（默认 opt-in）
  ├── my-plugin/
  │   ├── plugin.yaml
  │   ├── __init__.py
  │   ├── SKILL.md (可选)
  │   └── requirements.txt (可选)
  └── another-plugin/
      └── __init__.py

plugins/                        # 内置官方插件
  ├── context_engine/           # 上下文引擎插件
  ├── disk-cleanup/             # 磁盘清理
  ├── google_meet/              # Google Meet 集成
  ├── image_gen/                # 图像生成后端
  ├── memory/                   # 外部记忆后端
  ├── platforms/irc/            # 网关平台插件参考实现
  ├── observability/            # 可观测性
  ├── spotify/                  # Spotify 控制
  └── strike-freedom-cockpit/   # 仪表盘
```

插件发现来源：

| 来源 | 路径/机制 | 加载规则 |
|---|---|---|
| Bundled | 仓库 `plugins/` 或 `HERMES_BUNDLED_PLUGINS` | `backend` 和 `platform` 自动加载 |
| User | `get_hermes_home()/plugins` | 通过 `plugins.enabled` 显式启用 |
| Project | 当前项目 `.hermes/plugins` | 需 `HERMES_ENABLE_PROJECT_PLUGINS`，再显式启用 |
| Pip | `hermes_agent.plugins` entry point | 显式启用 |

目录插件需要 `plugin.yaml` manifest 和带 `register(ctx)` 的 `__init__.py`。Manifest 的 `kind` 支持 `standalone`、`backend`、`exclusive`、`platform`。


### 2.4 上下文引擎插件 (plugins/context_engine/)

允许用户为 Agent 提供自定义的提示词片段：

```python
def on_context_assembly(current_prompt):
    """在提示词组装时注入自定义内容"""
    return current_prompt + "\n[Custom rules from my plugin...]"
```

### 2.5 平台插件

网关平台也可以作为插件注册，不需要修改核心平台枚举或 adapter factory：

```python
def register(ctx):
    ctx.register_platform(
        name="irc",
        label="IRC",
        adapter_factory=lambda cfg: IRCAdapter(cfg),
        check_fn=check_requirements,
        validate_config=validate_config,
        is_connected=is_connected,
        required_env=["IRC_SERVER", "IRC_CHANNEL", "IRC_NICKNAME"],
        setup_fn=interactive_setup,
        allowed_users_env="IRC_ALLOWED_USERS",
        allow_all_env="IRC_ALLOW_ALL_USERS",
        max_message_length=450,
        platform_hint="You are chatting via IRC. Use plain text only.",
    )
```

注册信息进入 `gateway.platform_registry.platform_registry`。`Platform("irc")` 会通过动态 `_missing_()` 创建稳定的插件平台枚举成员，`toolsets.py` 会为 `hermes-irc` 自动生成核心工具集并合并插件工具。

## 3. MCP (Model Context Protocol)

### 3.1 架构

```
AIAgent
  │
  ├── ToolRegistry
  │     └── mcp_tool.py (MCP 客户端)
  │           ├── MCP Server 1 (filesystem)
  │           ├── MCP Server 2 (puppeteer)
  │           ├── MCP Server 3 (database)
  │           └── MCP Server N (custom)
  │
  └── 工具调用 → registry.dispatch() → MCP handler
```

### 3.2 功能

| 功能 | 说明 |
|---|---|
| **工具发现** | `tools/list` 请求获取服务器提供的工具列表 |
| **工具执行** | `tools/call` 请求调用具体工具 |
| **动态更新** | `notifications/tools/list_changed` 触发工具重新发现 |
| **多发器管理** | 并发连接多个 MCP 服务器，每个独立管理 |
| **OAuth 支持** | `tools/mcp_oauth.py` + `tools/mcp_oauth_manager.py` 管理授权 |

### 3.3 动态注册/注销

```python
# MCP 工具的热注册
for tool in mcp_server_tools:
    registry.register(
        name=f"mcp-{tool.name}",
        toolset=f"mcp-{server_name}",
        schema=tool.inputSchema,
        handler=create_mcp_handler(server_name, tool.name),
    )

# 服务器刷新时的热注销
for tool in old_tools:
    registry.deregister(tool.name)
```

### 3.4 配置

```yaml
# ~/.hermes/config.yaml
mcp:
  servers:
    filesystem:
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed"]
    puppeteer:
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-puppeteer"]
```

## 4. ACP (Agent Communication Protocol)

### 4.1 什么是 ACP

ACP 是 **代理-编辑器通信协议**（`agent-client-protocol`），允许 Hermes Agent 与 IDE/编辑器深度集成：

```
IDE (VS Code / JetBrains)
  │ ←── ACP Protocol ──→ │
  │                    Hermes ACP Server
  │                    (acp_adapter/server.py ~38K)
  │                         │
  │                    AIAgent 实例
```

### 4.2 ACP Server 功能

| 功能 | 说明 |
|---|---|
| **会话管理** | IDE 可以创建/切换/关闭 Agent 会话 |
| **文件变更推送** | Agent 的文件修改实时同步到编辑器 |
| **诊断信息** | 获取编译错误、Lint 警告等 |
| **选区操作** | 对编辑器选中的代码片段操作 |
| **权限授理** | IDE 可以管理 Agent 的命令审批 |
| **认证** | `acp_adapter/auth.py` 提供安全认证 |

### 4.3 ACP 注册中心 (acp_registry/)

定义 Agent 元数据和图标，使其能被 ACP 兼容的编辑器发现和连接。

```json
// acp_registry/agent.json
{
    "name": "Hermes Agent",
    "version": "0.11.0",
    "capabilities": ["terminal", "file_ops", "browser", ...]
}
```

## 5. 三种扩展方式的比较

| 维度 | 插件 | MCP | ACP |
|---|---|---|---|
| **目的** | 行为扩展（钩子） | 工具扩展（能力） | IDE 集成 |
| **开发语言** | Python | 任意 | 任意（通过 stdio/HTTP） |
| **安装** | 本地目录 | 配置文件指定命令 | 编辑器配置 |
| **隔离性** | 进程内（同 Python 进程） | 独立进程 | 独立进程 |
| **典型场景** | 自定义审批逻辑、日志钩子 | 数据库查询、API 调用 | IDE 内使用 Agent |
