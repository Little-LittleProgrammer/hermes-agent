# 开发入门与调试指南

## 1. 快速开始

### 1.1 环境准备

```bash
# Python >= 3.11
python --version

# 安装依赖
pip install -e ".[dev,messaging,cli]"

# 或使用 uv (更快)
uv pip install -e ".[dev,messaging,cli]"
```

### 1.2 初始化

```bash
# 运行设置向导
python cli.py setup

# 或手动配置
mkdir -p ~/.hermes
cp cli-config.yaml.example ~/.hermes/config.yaml
# 编辑 config.yaml 填入 API 密钥
```

### 1.3 第一次运行

```bash
# 交互模式
python cli.py

# 单次查询
python cli.py -q "列出当前目录的文件"

# 列出可用工具
python cli.py --list-tools

# 只启用特定工具集
python cli.py --toolsets web,terminal
```

## 3. 关键设计模式

### 3.1 全局单例

| 单例 | 文件 | 用途 |
|---|---|---|
| `registry` | `tools/registry.py` | 全局工具注册中心 |
| `process_registry` | `tools/process_registry.py` | 全局进程管理器 |

### 3.2 策略模式 — Transport

AIAgent 不感知底层 LLM 提供商，由 transport 策略对象处理所有差异。

### 3.3 观察者模式 — 插件钩子

插件系统允许外部代码订阅 Agent 生命周期事件。

### 3.4 编排器模式 — MemoryManager

MemoryManager 编排多个 MemoryProvider，对上层提供统一接口。

### 3.5 模板方法 — BasePlatformAdapter

所有平台适配器的公共流程在基类定义，具体实现留给子类。

```json
    "justMyCode": false

```

### 4.3 关键调试点

| 想调试什么 | 看哪里 |
|---|---|
| LLM 请求/响应 | `run_agent.py` 中 `_stream_llm_call()` 附近的 logger.debug |
| 工具调用 | `tools/registry.py` 中 `dispatch()` 的异常捕获 |
| 提示词内容 | `agent/prompt_builder.py` 的各个 build_* 函数 |
| 记忆读写 | `agent/memory_manager.py` 的 prefetch_all/sync_all |
| 上下文压缩 | `agent/context_compressor.py` 的 compress() |
| 消息路由 | `gateway/stream_consumer.py` 的 feed/flush |

## 5. 测试

### 5.1 测试结构

```
tests/
  ├── test_*.py              # 单元测试
  ├── conftest.py            # pytest fixtures
  └── ...
```

### 5.2 运行测试

```bash
pytest tests/ -x -v
pytest tests/test_registry.py -x -v  # 只运行特定文件
```

### 5.3 测试注意事项

- 集成测试可能需要真实的 API 密钥
- 工具测试可能产生副作用（文件系统修改、网络请求）
- 使用 `pytest-xdist` 可以并行运行测试

## 6. 常见开发任务

### 6.1 添加新工具

1. 创建 `tools/xxx.py`（可参考已有的简单工具如 `tools/todo_tool.py`）
2. 在文件末尾调用 `registry.register(...)`
3. 在 `toolsets.py` 的合适 toolset 中添加工具名
4. 运行 `python cli.py --list-tools` 验证

### 6.2 添加新平台

1. 创建 `gateway/platforms/xxx.py`
2. 继承 `BasePlatformAdapter`
3. 实现 `start()`, `stop()`, `send_message()` 等方法
4. 在 `gateway/config.py` 添加平台配置

### 6.3 添加新模型提供商

1. 在 `agent/transports/` 创建新文件
2. 继承 `ProviderTransport`
3. 实现 4 个抽象方法
4. 在 `agent/model_metadata.py` 添加模型信息

## 7. 性能分析

### 7.1 Token 估算

```python
tokens = estimate_messages_tokens_rough(messages)
print(f"Estimated tokens: {tokens}")
```

### 7.2 Prompt 大小优化

- 控制技能索引的大小（禁用不需要的技能）
- 使用 Anthropic 的 prompt caching
- 定期清理过期的记忆文件

## 8. 常见问题

| 问题 | 解决方案 |
|---|---|
| `ImportError: No module named 'xxx'` | 安装对应的 optional dependency |
| SSL 证书错误 | 检查 `SSL_CERT_FILE` 环境变量 |
| API 调用超时 | 调整 `HERMES_PROVIDER_TIMEOUT` |
| 工具注册不上 | 检查 AST 是否有 `registry.register(...)` 顶层调用 |
| Gateway 内存过高 | 降低 `_AGENT_CACHE_MAX_SIZE` |
