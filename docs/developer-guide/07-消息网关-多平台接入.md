# 消息网关架构 — 多平台接入

## 1. 设计理念

Hermes Agent 的网关层实现了**一个核心、多端接入**的架构。用户可以从手机、电脑、任何消息平台向代理发消息，代理在远程服务器/容器中执行任务，然后将结果返回给用户。整个流程对代理核心完全透明——它不感知消息来自哪个平台。

### 3.1 生命周期管理

```python
class GatewayRunner:

        self.session_manager = SessionManager()

    async def start(self):
        # 1. 初始化 Session Manager (SQLite)
        # 2. 根据配置启动启用的平台适配器
        # 3. 启动流消费者
        # 4. 启动投递路由器的定时任务
        # 5. 启动 Agent 缓存清理器
```

### 3.2 Agent 缓存

在 Gateway 中，AIAgent 实例是昂贵的资源（LLM 客户端、工具 schema、记忆提供商），不能每个消息都新建。缓存策略：

```
最大缓存: 128 个实例
驱逐策略: LRU + 空闲 TTL (1小时)
清理器: 每 5 分钟运行一次

_session_expiry_watcher():
    for each cached agent:
        if agent.idle_time > 1h:
            evict(agent)
```

### 3.3 消息处理流程

```
用户发送消息到 Telegram
  │
  ├─ TelegramAdapter.receive_message(raw_update)
  │   └─ 解析 → 标准化为 Hermes 内部格式
  │       {session_key, user_id, text, attachments, ...}
  │
  ├─ GatewayRunner.route_message(msg)
  │   ├── 查找/创建 Session
  │   │     └─ session_manager.get_or_create(session_key)
  │   ├── 获取/创建 AIAgent
  │   │     └─ _get_or_create_agent(session_key, user_config)
  │   └── 提交到线程池执行
  │         └─ executor.submit(agent.run_conversation, msg.text)
  │
  ├─ 流式响应
  │   └─ stream_consumer.feed(session_key, delta_chunks)
  │       └─ 通过 delivery_router 投递到正确平台/频道
  │
  └─ TelegramAdapter.send_message(chat_id, response)
       └─ 处理 Markdown 格式、代码块、媒体文件
```

## 4. Session Manager (gateway/session.py ~55K)

### 4.1 存储结构 (SQLite WAL + FTS5)

```sql
CREATE TABLE sessions (
    id TEXT PRIMARY KEY,        -- 会话 ID
    platform TEXT,               -- 平台名称
    user_id TEXT,                -- 用户 ID
    title TEXT,                  -- 会话标题（AI 生成）
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    metadata JSON
);

CREATE VIRTUAL TABLE session_messages_fts USING fts5(
    content,                     -- 消息内容全文索引
    session_id UNINDEXED,
    tokenize='porter unicode61'
);
```

### 4.2 关键功能

| 功能 | 说明 |
|---|---|
| **会话隔离** | 每个用户-平台组合可以有多个会话，互不干扰 |
| **FTS5 全文搜索** | Agent 可以搜索自己的历史对话 |
| **标题生成** | `agent/title_generator.py` 用辅助模型自动生成会话标题 |
| **消息重放** | 切换模型/重置上下文时可以重放历史消息 |
| **会话迁移** | 支持将会话导出/导入（`claw migrate` 命令） |

## 5. Stream Consumer (gateway/stream_consumer.py ~47K)

负责将 AIAgent 的流式输出分发到正确的目标：

```python
class StreamConsumer:
    def feed(self, session_key, delta):
        """接收 delta chunk，累积并批量发送"""
        self.buffers[session_key].append(delta)
        if self._should_flush(session_key):
            self._flush(session_key)
    
    def _flush(self, session_key):
        """将缓冲区内容发送到对应的平台适配器"""
        text = "".join(self.buffers[session_key])
        adapter = self._get_adapter_for_session(session_key)
        adapter.send_stream_update(chat_id, text)
```

**批量刷新策略**：不是每个 token 立即发送，而是基于时间（~200ms）或换行符触发刷新，减少 API 调用次数。

## 6. 平台适配器

### 6.1 抽象基类

```python
class BasePlatformAdapter:
    platform_name: str                    # "telegram", "discord" 等
```

### 6.3 添加新平台的步骤

1. 创建 `gateway/platforms/new_platform.py`
2. 继承 `BasePlatformAdapter`，实现所有抽象方法
3. 在 `gateway/platforms/__init__.py` 注册
4. 在 `gateway/config.py` 添加平台配置项
5. 添加消息格式转换逻辑

## 8. 镜像模式 (gateway/mirror.py ~6K)

支持**跨平台会话镜像** — 同一个 AIAgent 的对话可以在多个平台之间同步：

```
用户 A 在 Telegram 上开始对话
  → 用户 B 在 Discord 上加入同一会话
  → 两人看到相同的 Agent 响应
  → 两人的消息都发送给同一个 AIAgent
```

## 9. 配对系统 (gateway/pairing.py ~12K)

安全的设备配对流程：

```
1. 用户运行 `hermes pair` 获取配对码
2. 在目标平台上发送配对指令
3. Gateway 验证配对码 → 建立绑定
4. 会话持久化，设备可安全断开
```
